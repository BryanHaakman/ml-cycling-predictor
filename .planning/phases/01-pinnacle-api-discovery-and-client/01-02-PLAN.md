Write data/odds.py with the following structure (follow RESEARCH.md §Architecture Patterns exactly):

```
Module docstring (per scraper.py style)

Imports: os, json, logging, re, dataclasses, typing, datetime, requests

Constants (UPPER_SNAKE_CASE):
  PINNACLE_API_BASE = "https://guest.api.arcadia.pinnacle.com/0.1"
  PINNACLE_CYCLING_SPORT_ID = 45
  REQUEST_TIMEOUT = 60
  KEY_CACHE_PATH = os.path.join(os.path.dirname(__file__), ".pinnacle_key_cache")
  ODDS_LOG_PATH = os.path.join(os.path.dirname(__file__), "odds_log.jsonl")
  PINNACLE_HOME_URL = "https://www.pinnacle.com/"

log = logging.getLogger(__name__)

class PinnacleAuthError(Exception):
    """Raised when X-Api-Key is missing, expired, or rejected."""

@dataclass
class OddsMarket:
    rider_a_name: str
    rider_b_name: str
    odds_a: float
    odds_b: float
    race_name: str
    matchup_id: str

def _american_to_decimal(american: int) -> float:
    """Convert American odds to decimal. Private — do not import from models."""
    if american > 0:
        return round(american / 100.0 + 1.0, 4)
    return round(100.0 / abs(american) + 1.0, 4)

def _extract_key_from_bundle() -> Optional[str]:
    """Fetch Pinnacle's main JS bundle and regex-extract the 32-char API key.

    Method:
    1. GET https://www.pinnacle.com/ HTML
    2. Find the main JS bundle: regex r'src="(/en/sports/[^"]*\.js)"' or the
       largest versioned /static/js/main.[hash].chunk.js src attribute
    3. GET https://www.pinnacle.com{bundle_path}
    4. Regex-match for API key pattern: r'"X-Api-Key"\s*:\s*"([A-Za-z0-9]{32})"'
       OR r'apiKey["\s:=]+([A-Za-z0-9]{32})' (try both patterns)
    5. Return the first match, or None if not found

    Log a warning on failure; do not raise — caller handles the None case.
    Set timeout=REQUEST_TIMEOUT. Use requests (not cloudscraper — guest API needs no bypass).
    """

def _get_api_key() -> str:
    """Resolve the X-Api-Key using the lookup chain: env var -> cache -> JS bundle.

    Order (per D-13, D-14, D-16):
    1. Check os.environ.get("PINNACLE_API_KEY") — if set, return it immediately (D-16)
    2. Check KEY_CACHE_PATH — if file exists and is non-empty, return stripped contents (D-14)
    3. Call _extract_key_from_bundle() — if returns a key, write to KEY_CACHE_PATH, return key
    4. All paths exhausted → raise PinnacleAuthError(
           "Pinnacle API key could not be extracted. "
           "Set the PINNACLE_API_KEY environment variable as a manual override."
       )
    """

def _invalidate_key_cache() -> None:
    """Delete KEY_CACHE_PATH if it exists. Called on 401/403 before retry."""
    try:
        os.remove(KEY_CACHE_PATH)
    except FileNotFoundError:
        pass

def _check_auth(response: requests.Response) -> None:
    """Raise PinnacleAuthError on HTTP 401 or 403 (per D-15).

    Message must include the HTTP status and name the PINNACLE_API_KEY env var.
    Example: "Pinnacle API key is expired or invalid (HTTP 403).
              Set PINNACLE_API_KEY environment variable as a manual override."
    """

def _append_audit_log(
    markets: list[OddsMarket],
    fetch_status: str,
    error: Optional[str] = None,
) -> None:
    """Append one JSONL record to ODDS_LOG_PATH (per D-09, D-10).

    Record fields:
    - fetched_at: datetime.utcnow().isoformat() + "Z"
    - status: "ok" | "empty" | "auth_error" | "error"
    - market_count: len(markets)
    - markets: [dataclasses.asdict(m) for m in markets]  # decimal odds, never American
    - error: str (only if error param is not None)

    Opens ODDS_LOG_PATH in "a" mode (creates file on first write).
    """

def fetch_cycling_h2h_markets() -> list[OddsMarket]:
    """Fetch today's cycling H2H matchups from Pinnacle's internal API.

    Auth retry logic (per D-15):
    1. Call _get_api_key() — raises PinnacleAuthError if all paths exhausted
    2. Make API requests; on 401/403: invalidate cache, call _get_api_key() again
    3. If second attempt also 401/403: raise PinnacleAuthError

    Fetch flow:
    1. GET /sports/45/leagues?all=false → list of active cycling leagues
    2. For each league:
       a. GET /leagues/{id}/matchups → list of matchup dicts
       b. GET /leagues/{id}/markets/straight → list of market dicts
       c. Build market_by_id = {m["matchupId"]: m for m in markets_list}
       d. For each matchup: if market exists AND market["status"] == "open":
          build OddsMarket with decimal odds
    3. Call _append_audit_log(markets, "ok" if markets else "empty")
    4. Return markets (may be empty list — per D-11, never raise on empty)

    Guard against non-list responses (RESEARCH.md Pitfall 1):
    - Check isinstance(response.json(), list) before iterating
    - Log warning and skip the league (do not crash)

    Headers for every request:
    {
        "X-Api-Key": api_key,
        "Referer": "https://www.pinnacle.com/",
        "Accept": "application/json",
    }

    Timeout: REQUEST_TIMEOUT (60s) per request.
    On requests.RequestException (network error, timeout): log warning, call
    _append_audit_log([], "error", str(e)), raise (do not swallow network errors).
    """
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| data/odds.py → Pinnacle API | All HTTP responses from guest.api.arcadia.pinnacle.com are untrusted; response shape may change without notice |
| Pinnacle JS bundle → key extraction | Bundle content is third-party; regex must be defensive against format changes |
| data/.pinnacle_key_cache → disk | File contains a credential-like token; gitignored to prevent accidental commit |
| PINNACLE_API_KEY env var → process | Env var bypasses extraction; must not be logged in plaintext |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | data/.pinnacle_key_cache | mitigate | Added to .gitignore in Task 1 Step C. File is plain text; world-readable on disk but not committed. Acceptable for personal tool. |
| T-02-02 | Information Disclosure | PINNACLE_API_KEY in audit log | mitigate | _append_audit_log() records only market data, timestamps, and status. The API key MUST NOT appear in odds_log.jsonl. Verify: `grep -i "api.key\|x-api-key" data/odds_log.jsonl` should return 0 matches. |
| T-02-03 | Tampering | data/odds_log.jsonl | accept | Append-only JSONL per project convention (like data/bets.csv). No integrity check required — personal tool, no external verifier. |
| T-02-04 | Spoofing | Pinnacle JS bundle source | accept | Bundle is fetched over HTTPS from www.pinnacle.com. MitM attack would require a compromised CA. Beyond threat model for this tool. |
| T-02-05 | Elevation of Privilege | PinnacleAuthError message | mitigate | Error message names the env var to SET but does not print the current (bad) key value. Pattern: "Set PINNACLE_API_KEY env var" — not "Your key {key} is invalid". |
| T-02-06 | Denial of Service | Retry loop on 401/403 | mitigate | Auth retry is bounded to exactly ONE retry (invalidate cache → re-extract → if still 401/403 → raise). No infinite loop. |
</threat_model>
