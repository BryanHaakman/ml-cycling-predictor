# Requirements Archive: PaceIQ v1.0 — Pinnacle Preload

**Archived:** 2026-04-18
**Status:** All 15 v1 requirements complete

## v1 Requirements (All Complete)

### Odds Ingestion

- [x] **ODDS-01**: System fetches today's H2H cycling markets from Pinnacle's internal API using a stored session cookie — *Complete (Phase 1, pivoted to guest API in Phase 04.1)*
- [x] **ODDS-02**: Raw odds data is appended to an audit log (`data/odds_log.jsonl`) after each successful fetch — *Complete (Phase 1)*
- [x] **ODDS-03**: System shows a clear, actionable error message (including which env var to update) when the Pinnacle session cookie is expired or invalid — *Complete (Phase 1, env var reference removed in 04.1 guest pivot)*
- [x] **ODDS-04**: System can re-fetch Pinnacle odds independently without re-loading stage context or re-resolving rider names — *Complete (Phase 4)*

### Name Resolution

- [x] **NAME-01**: System resolves Pinnacle display names to PCS rider URLs via exact match against `cache.db` riders — *Complete (Phase 2)*
- [x] **NAME-02**: System resolves names that differ only by accents, special characters, or casing via unicode normalization before fuzzy matching — *Complete (Phase 2)*
- [x] **NAME-03**: System resolves ambiguous names via fuzzy matching (rapidfuzz); auto-accepts matches above confidence threshold without user input — *Complete (Phase 2)*
- [x] **NAME-04**: Confirmed name-to-PCS URL mappings are cached persistently in `data/name_mappings.json` and used on future runs before fuzzy matching — *Complete (Phase 2)*
- [x] **NAME-05**: Pairs where one or both riders could not be resolved are displayed in the UI with a manual rider search so the user can complete the match — *Complete (Phase 5)*

### Stage Context

- [x] **STGE-01**: System fetches stage details (distance, elevation gain, climb counts/categories, race tier, stage type, profile icon) from PCS via the `procyclingstats` lib given a Pinnacle race name — *Complete (Phase 3)*
- [x] **STGE-02**: Stage context fetch failure degrades gracefully — manual input fields remain available and prediction is not blocked — *Complete (Phase 3)*

### Batch Prediction UI

- [x] **UI-01**: User can click "Load from Pinnacle" in the batch H2H prediction UI to fetch today's available cycling markets — *Complete (Phase 5)*
- [x] **UI-02**: User can select a race from the fetched Pinnacle markets; selecting a race auto-populates all stage fields and all H2H pairs with odds — *Complete (Phase 5)*
- [x] **UI-03**: All auto-populated fields (stage details, rider selections, odds) remain individually editable before running predictions — *Complete (Phase 5)*
- [x] **UI-04**: User can click "Refresh Odds" to re-fetch current Pinnacle odds and update odds fields in an already-loaded session without clearing stage context or rider selections — *Complete (Phase 5)*
