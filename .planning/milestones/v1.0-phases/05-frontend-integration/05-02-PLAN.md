```javascript
async function refreshOdds() {
    if (!_pinnacleMatchupIds || _pinnacleMatchupIds.size === 0) return;

    const btn = document.getElementById('refresh-odds-btn');
    const err = document.getElementById('batch-error');
    err.style.display = 'none';

    // Loading state
    btn.disabled = true;
    btn.textContent = '\u23F3 Refreshing...';

    try {
        const ids = Array.from(_pinnacleMatchupIds);
        const res = await fetch('/api/pinnacle/refresh-odds', {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify({matchup_ids: ids})
        });
        const data = await res.json();

        if (!res.ok) {
            if (data.type === 'auth_error') {
                const envVar = data.env_var || 'PINNACLE_SESSION_COOKIE';
                err.innerHTML = 'Session expired. To fix:<br>' +
                    '1. Open DevTools on pinnacle.ca \u2192 Network tab \u2192 any request<br>' +
                    '2. Copy the X-Session header value<br>' +
                    '3. Set ' + envVar + '=&lt;value&gt; in your environment<br>' +
                    '4. Restart Flask (python webapp/app.py)';
            } else {
                err.textContent = 'Refresh failed: ' + (data.error || 'Unknown error');
            }
            err.style.display = 'block';
            return;
        }

        // Per D-13 + RESEARCH.md Pattern 4: DOM-based lookup via data-matchup-id attribute
        // Do NOT use positional array index -- row removal would cause index mismatch
        if (data.pairs) {
            // Build lookup from response: matchup_id -> updated odds
            const oddsById = {};
            data.pairs.forEach(pair => { oddsById[pair.matchup_id] = pair; });

            // Walk all pair rows in the DOM, update only those with matching matchup_id
            document.querySelectorAll('#batch-pairs-container .batch-pair-row').forEach(row => {
                const mid = row.getAttribute('data-matchup-id');
                if (!mid) return; // manually-added rows have no matchup_id -- skip
                const updated = oddsById[mid];
                if (!updated) return; // market closed or not in response -- skip silently

                const oddsAEl = row.querySelector('[id^="bp-odds-a-"]');
                const oddsBEl = row.querySelector('[id^="bp-odds-b-"]');

                if (oddsAEl && oddsAEl.getAttribute('data-source') === 'auto') {
                    oddsAEl.value = updated.odds_a;
                }
                if (oddsBEl && oddsBEl.getAttribute('data-source') === 'auto') {
                    oddsBEl.value = updated.odds_b;
                }
            });
        }
    } catch (e) {
        err.textContent = 'Refresh failed: ' + e.message;
        err.style.display = 'block';
    } finally {
        btn.disabled = false;
        btn.textContent = 'Refresh Odds';
    }
}
```

Key behaviors per decisions:
- Per D-13: `getAttribute('data-source') === 'auto'` check before updating each odds field. Fields with `data-source='user'` (set by the oninput handler added in Plan 01 via addPinnaclePair) are skipped.
- Per RESEARCH.md Pattern 4: Row lookup uses `querySelectorAll('#batch-pairs-container .batch-pair-row')` and `getAttribute('data-matchup-id')` -- NOT a positional array index. This safely handles: (a) manually-added rows (no data-matchup-id, skipped), (b) removed rows (row element gone, not visited), (c) closed markets (not in response, skipped).
- Per D-03: Button guard at top returns early if `_pinnacleMatchupIds` is null or empty (uses `.size` since it is a Set).
- Per D-17/D-18: Error display reuses same `#batch-error` div and same auth error message format as loadFromPinnacle(). Non-auth errors use "Refresh failed: {error text}" per UI-SPEC copywriting contract.
- Button loading state matches the pattern: disable + text change + restore in finally block.

| Flask API -> Browser DOM | Refresh response odds values written to input .value (safe -- no HTML injection) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Tampering | matchup_ids sent to /refresh-odds | accept | IDs originated from trusted /load response stored in _pinnacleMatchupIds; server validates they exist in Pinnacle's current market data; worst case is a 400 or empty response |
| T-05-07 | Repudiation | Odds values changed by refresh | accept | Odds refresh is a user-initiated action (explicit button click); no audit trail needed for odds display changes (bets are logged separately) |
| T-05-08 | Tampering | data-source attribute manipulation | accept | User could modify data-source via DevTools to force refresh to overwrite their edits; this is their own browser and only affects their local view -- no security impact |
</threat_model>
