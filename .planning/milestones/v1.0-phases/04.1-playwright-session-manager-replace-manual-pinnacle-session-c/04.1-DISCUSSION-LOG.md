### Follow-up: TTL Implementation

| Option | Description | Selected |
|--------|-------------|----------|
| Check on trigger, refresh if stale | TTL check at button-press time, no background timer | ✓ |
| Background timer auto-refreshes | Periodic background refresh | |
| You decide | Claude picks simplest approach | |

**User's choice:** Check on trigger, refresh if stale

### Additional Context (free-text): Adaptive TTL
User described an adaptive TTL system:
- TTL starts at infinity (no proactive refresh initially)
- On 401/403 failure, TTL is updated to the session age at failure time (new known minimum expiry)
- Over time, TTL converges on real Pinnacle session duration
- TTL persisted to disk so it survives restarts and keeps learning
