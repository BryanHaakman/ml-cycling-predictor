`is_one_day_race` MUST be determined inside `_do_fetch` because `Race()` makes a network call that must be covered by the 5-second timeout. Never pass `is_one_day` as a parameter from outside the executor.

```python
def _do_fetch(race_url: str) -> StageContext:
  """Fetch stage details from PCS. Runs inside ThreadPoolExecutor."""
  from procyclingstats import Race, Stage

  race = Race(race_url)
  is_one_day = race.is_one_day_race()  # ALWAYS from Race -- Critical Finding #1

  uci_tour = ""
  try:
    uci_tour = race.uci_tour()
  except Exception:
    log.warning("_do_fetch: uci_tour() failed for %s", race_url)

  if is_one_day:
    stage_url = f"{race_url}/result"  # Per Critical Finding #4
  else:
    today_mmdd = _date.today().strftime('%m-%d')  # MM-DD format -- Critical Finding #3
    stages = race.stages()
    todays = next((s for s in stages if s['date'] == today_mmdd), None)
    if todays is None:
      log.warning("_do_fetch: no stage matching today %s in %s", today_mmdd, race_url)
      return _unresolved_context()
    stage_url = todays['stage_url']

  stage = Stage(stage_url)
  return StageContext(
    distance=stage.distance(),
    vertical_meters=stage.vertical_meters(),
    profile_icon=stage.profile_icon(),
    profile_score=stage.profile_score(),
    is_one_day_race=is_one_day,      # Sourced from Race.is_one_day_race() -- Critical Finding #1
    stage_type=stage.stage_type(),
    race_date=stage.date(),
    race_base_url=_extract_base_url(race_url),
    num_climbs=len(stage.climbs()),  # Per D-09
    avg_temperature=stage.avg_temperature(),
    uci_tour=uci_tour,
    is_resolved=True,
  )
```

**Implement `_fetch_with_timeout(race_url: str) -> StageContext`:**

```python
def _fetch_with_timeout(race_url: str) -> StageContext:
  """Wrap _do_fetch in a 5-second timeout via ThreadPoolExecutor."""
  with concurrent.futures.ThreadPoolExecutor(max_workers=1) as executor:  # New executor per call -- Critical Finding #7
    future = executor.submit(_do_fetch, race_url)
    try:
      return future.result(timeout=TIMEOUT_SECONDS)
    except concurrent.futures.TimeoutError:  # Explicit subclass -- Critical Finding #6
      log.warning("fetch_stage_context: PCS timed out after %ss for %s", TIMEOUT_SECONDS, race_url)
      return _unresolved_context()
    except Exception as exc:
      log.warning("fetch_stage_context: PCS fetch failed for %s: %s", race_url, exc)
      return _unresolved_context()
```

```python
def fetch_stage_context(pinnacle_race_name: str) -> StageContext:
  """
  Map a Pinnacle race name to PCS stage details.

  Args:
    pinnacle_race_name: Race name from Pinnacle (e.g., "Tour de Romandie - Stage 3").

  Returns:
    StageContext with is_resolved=True if successful, is_resolved=False on any failure.
  """
  race_name = _parse_race_name(pinnacle_race_name)
  race_url = _resolve_race_url(race_name)
  if not race_url:
    return _unresolved_context()  # Per D-03
  return _fetch_with_timeout(race_url)
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Injection | _resolve_race_url SQL query | mitigate | Use parameterized query `WHERE year = ?` -- never interpolate race_name into SQL. rapidfuzz operates on Python strings in memory, not SQL. |
| T-3-02 | Denial of Service | _do_fetch PCS timeout | mitigate | ThreadPoolExecutor with `future.result(timeout=5)` caps wall-clock to 5 seconds. New executor per call prevents thread pool exhaustion. |
| T-3-03 | Information Disclosure | Log output | accept | Logs contain race names and PCS URLs only -- no PII, no credentials. Acceptable for personal tool. |
| T-3-04 | Tampering | PCS response manipulation | accept | procyclingstats lib parses HTML; a compromised PCS page could return wrong data. Low risk -- PCS is the authoritative source and data is used for prediction, not security decisions. |
| T-3-05 | Spoofing | Pinnacle race name input | accept | Input is a string from Pinnacle API (fetched in Phase 1 with session cookie). No auth decision based on this input. Fuzzy match threshold (75) limits false matches. |
