# Milestone v1.0: Pinnacle Preload

**Status:** SHIPPED 2026-04-15
**Phases:** 1-5 (+ 04.1 inserted)
**Total Plans:** 11

## Overview

This milestone added a live data ingestion layer to the existing batch prediction UI. The work proceeded in a strict build order: discover the Pinnacle internal API endpoint first (the only true unknown), then build the name resolver and stage context fetcher (parallel development possible), wire all three together into two Flask endpoints, and finally surface everything in the frontend. Each phase delivered a self-contained, testable capability that fed directly into the next.

## Milestone Summary

**Decimal Phases:**
- Phase 04.1: Guest API Pivot (inserted after Phase 04 — Playwright session manager experiment failed, pivoted to zero-auth guest API)

**Key Decisions:**
- procyclingstats lib for stage context (not MCP) — self-contained, VPS-safe
- Manual "Load" button trigger, not auto-load on startup — cookie expiry safety
- Separate refresh-odds endpoint — avoid wasteful PCS re-fetch on odds check
