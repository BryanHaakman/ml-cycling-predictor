# Technology Stack
> Python ML pipeline with a Flask web frontend, SQLite persistence, and a GitHub Actions nightly data feed.

## Overview

The project is a pure Python codebase running on Python 3.11. It scrapes cycling race data, engineers features, trains multiple ML models, and serves predictions via a minimal Flask API. There are no JavaScript build tools or frontend frameworks — the web UI is plain HTML/CSS rendered by Jinja2 templates. All dependencies are declared in a single `requirements.txt`.

## Languages

**Primary:**
- Python 3.11 — all application code, data pipeline, ML training, and web server

**Templates:**
- Jinja2 HTML — `webapp/templates/` (4 templates: `index.html`, `admin.html`, `pnl.html`, `results.html`)

**No secondary languages** — no JavaScript bundler, no TypeScript, no CSS preprocessor.

## Runtime

**Environment:**
- Python 3.11 (pinned in `.github/workflows/nightly-pipeline.yml` via `actions/setup-python@v5`)
- Local dev uses a `.venv/` virtual environment (`python3 -m venv .venv`)

**Package Manager:**
- `pip` — no lockfile present; `requirements.txt` uses `>=` minimum-version constraints only

**Lockfile:** Not present — versions are unpinned minimums, which means installs are not fully reproducible

## Frameworks

**Web:**
- Flask `>=3.0.0` — `webapp/app.py` — serves the prediction UI, REST API, P&L tracker, admin panel
  - Uses `render_template`, `jsonify`, `request`, `Response` (SSE streaming for admin script output)
  - No Flask extensions (no Flask-SQLAlchemy, no Flask-Login, no Blueprints)

**Machine Learning:**
- scikit-learn `>=1.3.0` — `models/benchmark.py` — Logistic Regression, Random Forest, calibration (`CalibratedClassifierCV`), `StandardScaler`, metrics
- XGBoost `>=2.0.0` — `models/benchmark.py` — gradient boosted trees; requires `libomp` on macOS
- PyTorch `>=2.1.0` — `models/neural_net.py` — feed-forward neural net (`CyclingNet`) with BatchNorm, ReLU, Dropout, Sigmoid output; trained with Adam optimizer and early stopping

**Data:**
- pandas `>=2.0.0` — feature matrix construction, parquet cache I/O
- numpy `>=1.24.0` — numerical operations throughout ML pipeline
- joblib `>=1.3.0` — model serialization (`pickle`-compatible), parallel job support

## Key Dependencies

**Critical:**
- `procyclingstats>=0.2.0` — `data/scraper.py` — third-party scraping library wrapping ProCyclingStats.com; parses `Race`, `Stage`, `Rider` objects. Primary data source.
- `cloudscraper>=1.2.71` — `data/scraper.py` — bypasses Cloudflare bot protection on PCS calendar pages; used for dynamic race discovery
- `xgboost>=2.0.0` — `models/benchmark.py` — best-performing model class; has OpenMP dependency on macOS
- `torch>=2.1.0` — `models/neural_net.py` — neural network training; large dependency (~2 GB); conflicts with XGBoost OpenMP on macOS (worked around with lazy imports)

**Infrastructure:**
- `requests>=2.31.0` — HTTP requests (available but scraping uses `cloudscraper` and `urllib.request` directly)
- `tqdm>=4.66.0` — progress bars in scraper and feature pipeline loops
- `matplotlib>=3.8.0` — `scripts/simulate_pnl.py` and experiment visualisations

## Data Storage

**Primary database:**
- SQLite via Python stdlib `sqlite3` — `data/cache.db` (gitignored)
- WAL mode enabled (`PRAGMA journal_mode=WAL`)
- Schema managed in `data/scraper.py` (`_create_tables`) and `data/pnl.py` (`_create_pnl_tables`)

**Feature cache:**
- Apache Parquet via pandas — `data/rider_features_cache.parquet`, `data/race_features_cache.parquet` (gitignored, regenerated by `scripts/precompute_features.py`)

**Trained models:**
- `models/trained/` — pickle files for each trained model + `benchmark_results.csv` (gitignored, regenerated by `scripts/train.py`)

## Configuration

**Environment:**
- No `.env` file present; no environment variable configuration detected
- No secrets required — PCS is a public website; no API keys used
- The web app runs with `debug=True, port=5001` when executed directly; no production config file

**Build:**
- No build system — plain `pip install -r requirements.txt`
- macOS only: `brew install libomp` required before XGBoost installs correctly

## Platform Requirements

**Development:**
- Python 3.11+
- macOS note: `brew install libomp` required for XGBoost; `caffeinate -s` used in training script to prevent sleep
- Linux: no extra system dependencies (CI runs `ubuntu-latest`)

**Production:**
- No deployment config detected (no Dockerfile, no Procfile, no `gunicorn`/`uvicorn` config)
- Flask dev server (`app.run(debug=True, port=5001)`) is the only run configuration present

## Scripts Inventory

All runnable scripts live in `scripts/`:

| Script | Purpose |
|--------|---------|
