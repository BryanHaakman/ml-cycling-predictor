# Technology Stack
> Python ML pipeline with a Flask web frontend, SQLite persistence, and a GitHub Actions nightly data feed.

## Overview

The project is a pure Python codebase running on Python 3.11. It scrapes cycling race data, engineers features, trains multiple ML models, and serves predictions via a minimal Flask API. There are no JavaScript build tools or frontend frameworks ‚Äî the web UI is plain HTML/CSS rendered by Jinja2 templates. All dependencies are declared in a single `requirements.txt`.

## Languages

**Primary:**
- Python 3.11 ‚Äî all application code, data pipeline, ML training, and web server

**Templates:**
- Jinja2 HTML ‚Äî `webapp/templates/` (4 templates: `index.html`, `admin.html`, `pnl.html`, `results.html`)

**No secondary languages** ‚Äî no JavaScript bundler, no TypeScript, no CSS preprocessor.

## Runtime

**Environment:**
- Python 3.11 (pinned in `.github/workflows/nightly-pipeline.yml` via `actions/setup-python@v5`)
- Local dev uses a `.venv/` virtual environment (`python3 -m venv .venv`)

**Package Manager:**
- `pip` ‚Äî no lockfile present; `requirements.txt` uses `>=` minimum-version constraints only

**Lockfile:** Not present ‚Äî versions are unpinned minimums, which means installs are not fully reproducible

## Frameworks

**Web:**
- Flask `>=3.0.0` ‚Äî `webapp/app.py` ‚Äî serves the prediction UI, REST API, P&L tracker, admin panel
  - Uses `render_template`, `jsonify`, `request`, `Response` (SSE streaming for admin script output)
  - No Flask extensions (no Flask-SQLAlchemy, no Flask-Login, no Blueprints)

**Machine Learning:**
- scikit-learn `>=1.3.0` ‚Äî `models/benchmark.py` ‚Äî Logistic Regression, Random Forest, calibration (`CalibratedClassifierCV`), `StandardScaler`, metrics
- XGBoost `>=2.0.0` ‚Äî `models/benchmark.py` ‚Äî gradient boosted trees; requires `libomp` on macOS
- PyTorch `>=2.1.0` ‚Äî `models/neural_net.py` ‚Äî feed-forward neural net (`CyclingNet`) with BatchNorm, ReLU, Dropout, Sigmoid output; trained with Adam optimizer and early stopping

**Data:**
- pandas `>=2.0.0` ‚Äî feature matrix construction, parquet cache I/O
- numpy `>=1.24.0` ‚Äî numerical operations throughout ML pipeline
- joblib `>=1.3.0` ‚Äî model serialization (`pickle`-compatible), parallel job support

## Key Dependencies

**Critical:**
- `procyclingstats>=0.2.0` ‚Äî `data/scraper.py` ‚Äî third-party scraping library wrapping ProCyclingStats.com; parses `Race`, `Stage`, `Rider` objects. Primary data source.
- `cloudscraper>=1.2.71` ‚Äî `data/scraper.py` ‚Äî bypasses Cloudflare bot protection on PCS calendar pages; used for dynamic race discovery
- `geopy>=2.4` ‚Äî `scripts/fetch_weather.py` ‚Äî geocodes race departure cities via Nominatim (OpenStreetMap); no API key required
- `xgboost>=2.0.0` ‚Äî `models/benchmark.py` ‚Äî best-performing model class; has OpenMP dependency on macOS
- `torch>=2.1.0` ‚Äî `models/neural_net.py` ‚Äî neural network training; large dependency (~2 GB); conflicts with XGBoost OpenMP on macOS (worked around with lazy imports)

**Infrastructure:**
- `requests>=2.31.0` ‚Äî HTTP requests (available but weather fetching uses `urllib.request` directly)
- `tqdm>=4.66.0` ‚Äî progress bars in scraper and feature pipeline loops
- `matplotlib>=3.8.0` ‚Äî `scripts/simulate_pnl.py` and experiment visualisations

## Data Storage

**Primary database:**
- SQLite via Python stdlib `sqlite3` ‚Äî `data/cache.db` (gitignored)
- WAL mode enabled (`PRAGMA journal_mode=WAL`)
- Schema managed in `data/scraper.py` (`_create_tables`) and `data/pnl.py` (`_create_pnl_tables`)

**Snapshot transport:**
- `data/db_snapshot.sql.gz` ‚Äî gzip-compressed SQL dump committed to git; loaded/dumped by `scripts/load_db.py` and `scripts/dump_db.py`

**Feature cache:**
- Apache Parquet via pandas ‚Äî `data/rider_features_cache.parquet`, `data/race_features_cache.parquet` (gitignored, regenerated by `scripts/precompute_features.py`)

**Trained models:**
- `models/trained/` ‚Äî pickle files for each trained model + `benchmark_results.csv` (gitignored, regenerated by `scripts/train.py`)

## Configuration

**Environment:**
- No `.env` file present; no environment variable configuration detected
- No secrets required ‚Äî all external APIs used (Open-Meteo, Nominatim) are keyless

## Platform Requirements

**Development:**
- Python 3.11+
- macOS note: `brew install libomp` required for XGBoost; `caffeinate -s` used in training script to prevent sleep
- Linux: no extra system dependencies (CI runs `ubuntu-latest`)

**Production:**
- No deployment config detected (no Dockerfile, no Procfile, no `gunicorn`/`uvicorn` config)
- Flask dev server (`app.run(debug=True, port=5001)`) is the only run configuration present
